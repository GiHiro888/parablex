# parablex-research — 調査・技術選定・比較検討

> 用途非依存の調査スキル。
> 共通の型は [../DOCTRINE.md](../DOCTRINE.md) を継承：推奨先出し・根拠主義(URL/一次情報)・confidence・人間判断分離・**捏造/誇張禁止**。

# inputs
- 調査テーマ（技術選定 / 比較 / 動向 / 実現可能性 等）
- 制約（現行スタック・コスト・期限など、任意）

# outputs
- 調査レポート（推奨先出し・出典付き）
- 指示時のみVault保存：`30_Reviews/{週} weekly-scout.md` または `20_Knowledge/K-{topic}.md`
- 再利用知見は `#to-knowledge` 付与

# 出力構成
1. **結論**（推奨＋一行根拠）
2. **比較マトリクス**（選択肢 × 評価軸、各セルに出典）
3. **適用可否**（即採用 / 要検証 / 見送り）＋理由
4. **リスク・限界・反証**（強みだけでなく弱点も）
5. **人間の意思決定事項**（コスト/方針/優先度）
6. **出典一覧**（一次情報優先）＋各主張のconfidence

# 観点ルーブリック
- 出典品質：一次情報（公式doc/論文/一次データ）＞ 二次（技術ブログ）＞ 三次（まとめ記事）
- 評価軸例：導入コスト・学習コスト・保守性・性能・エコシステム・将来性・ライセンス
- confidence：high=一次情報で確認 / mid=複数二次情報が一致 / low=単一/推測
- 鮮度：情報の日付を明記。古い情報は「要再確認」フラグ

# 手順（Pseudo）
1. テーマ分解 → 評価軸を先に確定
2. 一次情報優先で収集、各主張に出典・日付・confidence付与
3. 比較マトリクス化、現行スタックへの適用可否を判定
4. 反証・限界を能動的に探索（確証バイアス回避）
5. 推奨を結論先出し、人間決定事項を分離
6. 指示あればVault保存・知見抽出

# 期待出力サンプル
```markdown
## 結論
- 推奨：A案（学習コスト最小・現行スタック適合）
## 比較マトリクス
| 選択肢 | 導入コスト | 保守性 | 将来性 | 出典 |
|---|---|---|---|---|
| A | 低 | 高 | 中 | 公式doc(2026) |
| B | 中 | 高 | 高 | 公式doc/ベンチ |
## 適用可否
- 即採用：A（要検証なし）
- 要検証：B（性能ベンチを自前で）
- 見送り：C（保守停止・confidence low）
## リスク・限界
- A：大規模時の性能未検証（confidence mid）
## 人間の意思決定事項
- 性能 vs 学習コストの優先度
```
