# checklist-common — 全言語共通チェックリスト

## A. セキュリティ
- [ ] 外部入力（ユーザー/ファイル/API/環境変数/DB）を信頼しない
- [ ] インジェクション対策：パラメタ化クエリ・エスケープ・許可リスト
- [ ] 認証/認可：操作ごとに権限確認、IDOR（他人資源参照）防止
- [ ] 秘匿情報：鍵/トークンのハードコード・ログ出力なし
- [ ] エラーで内部情報（スタック/パス/SQL）を露出しない

## B. データ整合性・堅牢性
- [ ] parse/IO/ネットワーク例外を捕捉、失敗を利用者に通知
- [ ] 破壊的操作前にバックアップ/確認、可能ならトランザクション
- [ ] 破損データは無断破棄せず隔離退避
- [ ] スキーマ変更時の後方/前方互換・移行手段

## C. 入力検証・境界値
- [ ] 型・範囲・形式・enum（許可リスト）検証
- [ ] 長さ上限、null/undefined/空、負数/巨大値/特殊値
- [ ] 文字エンコーディング・正規化

## D. 信頼性・性能・保守性
- [ ] リソース（ハンドル/接続/メモリ）の確実な解放
- [ ] N+1・非有界ループ/クエリの回避
- [ ] 冪等性・競合（再実行/同時実行）耐性
- [ ] 重複コード・デッドコード・根拠なきマジックナンバー
