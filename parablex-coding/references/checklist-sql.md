# checklist-sql — SQL / データベースチェックリスト

## セキュリティ
- [ ] SQLインジェクション：プレースホルダ／バインド変数のみ。文字列連結・動的SQLは原則禁止
- [ ] 動的に列名・テーブル名・ORDER BYを組む場合は許可リスト照合（バインド不可のため）
- [ ] 最小権限：アプリ用DBユーザーはDDLや他DB参照を持たない
- [ ] 機密列（パスワード・個人情報）の平文保存なし・適切な暗号化/ハッシュ

## 整合性・設計
- [ ] 一意制約・外部キー・NOT NULL・CHECK制約で不整合を防ぐ
- [ ] トランザクション境界が業務単位と一致（部分コミットで不整合化しない）
- [ ] 適切な分離レベル（ファントム/ダーティリード許容範囲の確認）
- [ ] マイグレーションの可逆性（down）・本番適用順序・後方互換

## パフォーマンス
- [ ] WHERE/JOIN/ORDER BY列のインデックス。関数適用列でのインデックス不使用に注意
- [ ] N+1（ループ内クエリ）回避＝JOIN/IN/一括取得
- [ ] 非有界SELECT（LIMITなし全件）・SELECT * の濫用回避
- [ ] 大量更新/削除のバッチ分割・ロック範囲

## 運用
- [ ] エラーでSQL文や内部構造を露出しない
- [ ] バックアップ・リストア手順の存在
- [ ] タイムゾーン・文字コード（utf8mb4等）・照合順序の一貫性
